# 🗄️ 数据库配置和问题解决指南

> 解决 "Cannot Get JDBC Connection" 错误

---

## ❌ 当前错误

```
CannotGetJdbcConnectionException: Failed to obtain JDBC Connection
Could not create connection to database server (115.190.125.177:3306)
```

**原因**：无法连接到远程 MySQL 数据库

---

## 🎯 解决方案（选择其一）

### 方案一：使用本地 MySQL（推荐）⭐

#### Step 1: 安装 MySQL（如果没有）

**Windows 用户**：

```powershell
# 使用 Chocolatey 安装
choco install mysql

# 或使用 winget 安装
winget install Oracle.MySQL
```

**或手动下载**：https://dev.mysql.com/downloads/mysql/

#### Step 2: 启动 MySQL 服务

```powershell
# 检查 MySQL 服务状态
net start | findstr MySQL

# 启动 MySQL 服务
net start MySQL80

# 如果服务名不同，查看服务列表
services.msc
```

#### Step 3: 创建数据库

```powershell
# 登录 MySQL
mysql -u root -p

# 输入密码后，执行以下 SQL
```

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS tibetan_medicine 
DEFAULT CHARACTER SET utf8mb4 
DEFAULT COLLATE utf8mb4_unicode_ci;

-- 查看数据库
SHOW DATABASES;

-- 退出
EXIT;
```

#### Step 4: 导入数据表

```powershell
# 在 backend 目录下执行
cd e:\project\uniapp-lr\backend

# 导入表结构
mysql -u root -p tibetan_medicine < database/schema.sql

# 导入初始数据
mysql -u root -p tibetan_medicine < database/init_data.sql
```

#### Step 5: 修改配置使用本地数据库

**方法 A：修改 application.yml（推荐）**

打开 `src/main/resources/application.yml`，修改第 13-15 行：

```yaml
datasource:
  url: jdbc:mysql://localhost:3306/tibetan_medicine?useUnicode=true&characterEncoding=utf8mb4&serverTimezone=Asia/Shanghai
  username: root
  password: 你的MySQL密码
```

**方法 B：使用配置文件切换**

我已经创建了 `application-local.yml`，在启动时指定：

```bash
# Maven 启动
mvn spring-boot:run -Dspring.profiles.active=local

# 或在 IDEA 中
Run > Edit Configurations > Active profiles: local
```

#### Step 6: 重启项目

在 IDEA 中：
- 停止当前运行
- 按 **Shift + F10** 重新运行

### 方案二：检查远程数据库连接

如果必须使用远程数据库（`115.190.125.177`），请检查：

#### 1. 网络连通性

```powershell
# 测试能否连接到服务器
ping 115.190.125.177

# 测试端口是否开放
Test-NetConnection -ComputerName 115.190.125.177 -Port 3306
```

#### 2. MySQL 是否运行

联系服务器管理员确认：
- MySQL 服务是否运行
- 防火墙是否开放 3306 端口

#### 3. 用户权限

确认用户 `tibetan_app` 是否有远程连接权限：

```sql
-- 在服务器上执行
SELECT Host, User FROM mysql.user WHERE User='tibetan_app';

-- 如果没有远程权限，授予权限
GRANT ALL PRIVILEGES ON tibetan_medicine.* TO 'tibetan_app'@'%' IDENTIFIED BY 'LR-project-7788';
FLUSH PRIVILEGES;
```

#### 4. 测试连接

```powershell
# 使用命令行测试
mysql -h 115.190.125.177 -P 3306 -u tibetan_app -p
# 输入密码：LR-project-7788
```

### 方案三：使用 H2 内存数据库（快速测试）

如果只是想快速测试 API，可以临时使用 H2 内存数据库。

#### 1. 添加 H2 依赖

在 `pom.xml` 中添加：

```xml
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>runtime</scope>
</dependency>
```

#### 2. 修改配置

```yaml
spring:
  datasource:
    driver-class-name: org.h2.Driver
    url: jdbc:h2:mem:testdb
    username: sa
    password: 
  h2:
    console:
      enabled: true
```

**注意**：H2 是内存数据库，重启后数据会丢失。

## 🐛 常见问题

### Q1: MySQL 服务启动失败

**解决**：
```powershell
# 查看错误日志
Get-EventLog -LogName Application -Source MySQL -Newest 10

# 或检查配置文件
notepad "C:\ProgramData\MySQL\MySQL Server 8.0\my.ini"
```

### Q2: Access denied for user 'root'

**解决**：
```powershell
# 重置 MySQL 密码
# 1. 停止 MySQL 服务
net stop MySQL80

# 2. 跳过密码验证启动
mysqld --console --skip-grant-tables

# 3. 在新窗口重置密码
mysql -u root
ALTER USER 'root'@'localhost' IDENTIFIED BY '新密码';
FLUSH PRIVILEGES;
```

### Q3: 表不存在

**解决**：
```powershell
# 重新导入数据库脚本
mysql -u root -p tibetan_medicine < database/schema.sql
mysql -u root -p tibetan_medicine < database/init_data.sql
```

### Q4: 时区错误

**解决**：在连接 URL 中添加时区参数（已包含）：
```
?serverTimezone=Asia/Shanghai
```

## 📝 推荐配置（本地开发）

```yaml
# application.yml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/tibetan_medicine?useUnicode=true&characterEncoding=utf8mb4&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
    username: root
    password: 你的密码
```

**参数说明**：
- `useUnicode=true` - 使用 Unicode 编码
- `characterEncoding=utf8mb4` - 支持 emoji 和特殊字符
- `serverTimezone=Asia/Shanghai` - 设置时区
- `allowPublicKeyRetrieval=true` - 允许公钥检索（MySQL 8.0+）
