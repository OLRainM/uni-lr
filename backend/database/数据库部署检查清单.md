# 数据库部署检查清单

> 快速检查数据库是否部署成功

**执行时间**：3-5分钟  
**适用场景**：数据库部署后的验证

---

## ✅ 快速验证脚本

将以下SQL复制到MySQL命令行执行：

```sql
-- ====================================
-- 数据库部署验证脚本
-- ====================================

-- 1. 检查数据库是否存在
SELECT SCHEMA_NAME 
FROM information_schema.SCHEMATA 
WHERE SCHEMA_NAME = 'tibetan_medicine';
-- 预期：返回1行记录

-- 2. 检查字符集
SELECT DEFAULT_CHARACTER_SET_NAME, DEFAULT_COLLATION_NAME 
FROM information_schema.SCHEMATA 
WHERE SCHEMA_NAME = 'tibetan_medicine';
-- 预期：utf8mb4, utf8mb4_unicode_ci

USE tibetan_medicine;

-- 3. 检查表数量
SELECT COUNT(*) AS table_count 
FROM information_schema.TABLES 
WHERE TABLE_SCHEMA = 'tibetan_medicine';
-- 预期：12

-- 4. 列出所有表
SELECT TABLE_NAME 
FROM information_schema.TABLES 
WHERE TABLE_SCHEMA = 'tibetan_medicine'
ORDER BY TABLE_NAME;
-- 预期：12张表按字母顺序排列

-- 5. 检查每张表的记录数
SELECT 
  TABLE_NAME,
  TABLE_ROWS AS record_count
FROM information_schema.TABLES
WHERE TABLE_SCHEMA = 'tibetan_medicine'
ORDER BY TABLE_NAME;

-- 6. 验证必要的初始数据
-- 检查管理员
SELECT COUNT(*) AS admin_count FROM admins;
-- 预期：至少1个

-- 检查商品分类
SELECT COUNT(*) AS category_count FROM categories;
-- 预期：5个（如果执行了schema.sql）

-- 检查测试商品（如果执行了init_data.sql）
SELECT COUNT(*) AS product_count FROM products;
-- 预期：5个

-- 7. 验证管理员账号
SELECT id, username, nickname, role, status 
FROM admins 
WHERE username = 'admin';
-- 预期：1个超级管理员账号

-- 8. 检查索引
SELECT 
  TABLE_NAME,
  COUNT(DISTINCT INDEX_NAME) AS index_count
FROM information_schema.STATISTICS
WHERE TABLE_SCHEMA = 'tibetan_medicine'
GROUP BY TABLE_NAME
ORDER BY TABLE_NAME;

-- 9. 验证表引擎
SELECT 
  TABLE_NAME,
  ENGINE
FROM information_schema.TABLES
WHERE TABLE_SCHEMA = 'tibetan_medicine'
ORDER BY TABLE_NAME;
-- 预期：所有表都是InnoDB

-- 10. 总体报告
SELECT 
  '数据库' AS item,
  'tibetan_medicine' AS name,
  'utf8mb4' AS charset,
  '12' AS table_count,
  'InnoDB' AS engine
UNION ALL
SELECT 
  '表',
  TABLE_NAME,
  TABLE_COLLATION,
  TABLE_ROWS,
  ENGINE
FROM information_schema.TABLES
WHERE TABLE_SCHEMA = 'tibetan_medicine'
ORDER BY item, name;
```

## ⚠️ 常见问题排查

### 问题1：数据库不存在

**症状**：
```
ERROR 1049 (42000): Unknown database 'tibetan_medicine'
```

**解决**：
```bash
# 重新执行schema.sql
mysql -u root -p < schema.sql
```

### 问题2：表数量不对

**症状**：表数量少于12张

**解决**：
```sql
-- 查看创建表时的错误
SHOW WARNINGS;

-- 检查是否有表创建失败
SHOW CREATE TABLE 表名;

-- 重新执行schema.sql
SOURCE /path/to/schema.sql;
```

### 问题3：没有初始数据

**症状**：表是空的

**解决**：
```bash
# 执行初始化数据脚本
mysql -u root -p tibetan_medicine < init_data.sql
```

### 问题4：字符集错误

**症状**：
```
DEFAULT_CHARACTER_SET_NAME: latin1
```

**解决**：
```sql
-- 修改数据库字符集
ALTER DATABASE tibetan_medicine 
  CHARACTER SET utf8mb4 
  COLLATE utf8mb4_unicode_ci;

-- 修改表字符集
ALTER TABLE 表名 
  CONVERT TO CHARACTER SET utf8mb4 
  COLLATE utf8mb4_unicode_ci;
```

### 问题5：应用无法连接

**症状**：Spring Boot启动报错

**解决**：
```sql
-- 1. 检查用户是否存在
SELECT user, host FROM mysql.user WHERE user = 'tibetan_app';

-- 2. 重新授权
GRANT ALL PRIVILEGES ON tibetan_medicine.* TO 'tibetan_app'@'%';
FLUSH PRIVILEGES;

-- 3. 检查防火墙
# Linux
sudo firewall-cmd --list-all

-- 4. 检查MySQL绑定地址
# 编辑 my.cnf
bind-address = 0.0.0.0

-- 5. 重启MySQL
sudo systemctl restart mysqld
```
