# 服务器数据库部署指南

> MySQL数据库创建和初始化完整操作步骤

**适用环境**：Linux服务器（CentOS/Ubuntu）或 Windows Server  
**MySQL版本**：8.0+  
**字符集**：utf8mb4  
**执行时间**：约10-15分钟

---

## 📋 前置检查清单

### 1. 确认MySQL已安装

```bash
# 检查MySQL版本
mysql --version

# 应该显示类似：mysql  Ver 8.0.x
```

### 2. 确认MySQL服务运行中

```bash
# Linux (CentOS/RHEL)
systemctl status mysqld

# Linux (Ubuntu/Debian)
systemctl status mysql

# Windows
# 在服务管理器中查看 MySQL 服务状态
```

### 3. 准备数据库文件

确保以下文件已上传到服务器：

```
backend/database/
├── schema.sql          # 数据库结构（必需）
├── init_data.sql       # 测试数据（可选）
└── 服务器数据库部署指南.md  # 本文档
```

## 🚀 部署步骤

### 方法一：命令行直接执行（推荐）

#### Step 1: 连接MySQL

```bash
# 使用root用户连接
mysql -u root -p

# 输入MySQL root密码
```

#### Step 2: 执行数据库结构脚本

```bash
# 在MySQL命令行中执行
source /path/to/backend/database/schema.sql;

# 或退出MySQL后使用命令
mysql -u root -p < /path/to/backend/database/schema.sql
```

**执行成功标志**：
```
Query OK, 1 row affected (0.02 sec)
Database changed
Query OK, 0 rows affected (0.05 sec)
...
```

#### Step 3: 执行测试数据脚本（开发/测试环境）

```bash
# 在MySQL命令行中执行
source /path/to/backend/database/init_data.sql;

# 或
mysql -u root -p tibetan_medicine < /path/to/backend/database/init_data.sql
```

#### Step 4: 验证数据库创建

```sql
-- 查看数据库
SHOW DATABASES LIKE 'tibetan_medicine';

-- 切换到数据库
USE tibetan_medicine;

-- 查看所有表
SHOW TABLES;

-- 应该显示12张表：
-- admins
-- addresses
-- banners
-- cart
-- categories
-- coupons
-- knowledge
-- order_items
-- orders
-- products
-- user_coupons
-- users

-- 查看商品数据
SELECT COUNT(*) FROM products;
-- 应该显示 5 (如果执行了init_data.sql)

-- 查看管理员
SELECT id, username, nickname, role FROM admins;
-- 应该显示默认管理员
```

### 方法二：使用单行命令（快速部署）

#### 完整部署（结构+数据）

```bash
# Linux/Mac
mysql -u root -p < backend/database/schema.sql && \
mysql -u root -p tibetan_medicine < backend/database/init_data.sql

# 提示输入密码后自动执行
```

#### 仅部署结构（生产环境）

```bash
mysql -u root -p < backend/database/schema.sql
```

## 🔐 创建应用数据库用户

**重要**：不建议使用root用户连接应用，应创建专用用户。

### Step 1: 创建数据库用户

```sql
-- 创建用户（替换YOUR_PASSWORD为强密码）
CREATE USER 'tibetan_app'@'%' IDENTIFIED BY 'YOUR_PASSWORD';

-- 如果仅本地访问，使用：
CREATE USER 'tibetan_app'@'localhost' IDENTIFIED BY 'YOUR_PASSWORD';
```

### Step 2: 授予权限

```sql
-- 授予数据库所有权限
GRANT ALL PRIVILEGES ON tibetan_medicine.* TO 'tibetan_app'@'%';

-- 刷新权限
FLUSH PRIVILEGES;
```

### Step 3: 验证用户

```bash
# 使用新用户登录测试
mysql -u tibetan_app -p

# 在MySQL中测试
USE tibetan_medicine;
SHOW TABLES;
```

### Step 4: 记录数据库连接信息

```yaml
数据库配置（用于application.yml）：
  host: 服务器IP或域名
  port: 3306
  database: tibetan_medicine
  username: tibetan_app
  password: YOUR_PASSWORD
```

## ✅ 部署验证清单

执行以下SQL验证部署是否成功：

```sql
-- 1. 检查数据库字符集
SELECT DEFAULT_CHARACTER_SET_NAME, DEFAULT_COLLATION_NAME 
FROM information_schema.SCHEMATA 
WHERE SCHEMA_NAME = 'tibetan_medicine';
-- 应该显示：utf8mb4, utf8mb4_unicode_ci

-- 2. 检查表数量
SELECT COUNT(*) AS table_count 
FROM information_schema.TABLES 
WHERE TABLE_SCHEMA = 'tibetan_medicine';
-- 应该显示：12

-- 3. 检查每个表的记录数
SELECT 
  TABLE_NAME,
  TABLE_ROWS
FROM information_schema.TABLES
WHERE TABLE_SCHEMA = 'tibetan_medicine'
ORDER BY TABLE_NAME;

-- 4. 检查索引
SELECT 
  TABLE_NAME,
  INDEX_NAME,
  COLUMN_NAME
FROM information_schema.STATISTICS
WHERE TABLE_SCHEMA = 'tibetan_medicine'
ORDER BY TABLE_NAME, INDEX_NAME;

-- 5. 测试管理员登录
SELECT id, username, nickname, role, status 
FROM admins 
WHERE username = 'admin';
-- 应该显示一条记录

-- 6. 测试商品数据
SELECT id, name, price, stock, sales, is_hot, is_new 
FROM products 
LIMIT 5;

-- 7. 测试分类数据
SELECT id, name, sort 
FROM categories 
ORDER BY sort;
```

## 🔧 常见问题和解决方案

### 1. 权限不足错误

**错误信息**：
```
ERROR 1044 (42000): Access denied for user 'xxx'@'localhost' to database 'tibetan_medicine'
```

**解决方案**：
```sql
-- 使用root用户重新授权
GRANT ALL PRIVILEGES ON tibetan_medicine.* TO 'user'@'host';
FLUSH PRIVILEGES;
```

### 2. 字符集问题

**错误信息**：
```
ERROR 1273 (HY000): Unknown collation: 'utf8mb4_unicode_ci'
```

**解决方案**：
```sql
-- 检查MySQL版本（需要5.5.3+）
SELECT VERSION();

-- 升级MySQL或使用utf8
-- 修改schema.sql中的COLLATE为utf8_general_ci
```

### 3. 数据库已存在

**错误信息**：
```
ERROR 1007 (HY000): Can't create database 'tibetan_medicine'; database exists
```

**解决方案**：
```sql
-- 方案1：删除重建（注意：会丢失所有数据）
DROP DATABASE tibetan_medicine;
SOURCE schema.sql;

-- 方案2：仅更新表结构
USE tibetan_medicine;
-- 手动执行需要的建表语句
```

### 4. 表已存在

**解决方案**：
```sql
-- 删除单个表
DROP TABLE IF EXISTS table_name;

-- 删除所有表重建
DROP DATABASE tibetan_medicine;
CREATE DATABASE tibetan_medicine CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 5. 外部无法连接

**问题**：应用无法连接数据库

**解决方案**：
```bash
# 1. 检查防火墙
sudo firewall-cmd --list-all
sudo firewall-cmd --add-port=3306/tcp --permanent
sudo firewall-cmd --reload

# 2. 检查MySQL绑定地址
# 编辑 /etc/my.cnf 或 /etc/mysql/mysql.conf.d/mysqld.cnf
bind-address = 0.0.0.0

# 3. 重启MySQL
sudo systemctl restart mysqld

# 4. 确认用户允许远程连接
CREATE USER 'user'@'%' IDENTIFIED BY 'password';
GRANT ALL PRIVILEGES ON tibetan_medicine.* TO 'user'@'%';
FLUSH PRIVILEGES;
```

## 🔒 安全建议

### 1. 修改默认管理员密码

```sql
-- 登录后立即修改
UPDATE admins 
SET password = '新的BCrypt加密密码' 
WHERE username = 'admin';

-- 注意：密码需要使用BCrypt加密，不能使用明文
```

### 2. 数据库用户权限最小化

```sql
-- 应用用户只需要CRUD权限
GRANT SELECT, INSERT, UPDATE, DELETE ON tibetan_medicine.* TO 'tibetan_app'@'%';

-- 不要授予 DROP, CREATE, ALTER 等权限
```

### 3. 定期备份

```bash
# 备份数据库
mysqldump -u root -p tibetan_medicine > backup_$(date +%Y%m%d).sql

# 或使用gzip压缩
mysqldump -u root -p tibetan_medicine | gzip > backup_$(date +%Y%m%d).sql.gz

# 恢复备份
mysql -u root -p tibetan_medicine < backup_20251202.sql
```

### 4. 设置复杂密码

```sql
-- 密码要求：
-- - 至少12位
-- - 包含大小写字母、数字、特殊字符
-- - 不使用常见密码

-- 生成随机密码（Linux）
openssl rand -base64 16
```

## 📝 Spring Boot 配置

部署完成后，在 `application.yml` 中配置：

```yaml
spring:
  datasource:
    url: jdbc:mysql://服务器IP:3306/tibetan_medicine?useUnicode=true&characterEncoding=utf8mb4&serverTimezone=Asia/Shanghai
    username: tibetan_app
    driver-class-name: com.mysql.cj.jdbc.Driver
    password: YOUR_PASSWORD

  # JPA配置
  jpa:
    hibernate:
      ddl-auto: none  # 重要：不自动修改表结构
    show-sql: false   # 生产环境关闭SQL日志
    properties:
      hibernate:
        dialect: org.hibernate.dialect.MySQL8Dialect
```
