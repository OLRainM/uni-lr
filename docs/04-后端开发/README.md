## 💡 核心概念

### 1. 架构演进

```
当前架构（本地存储）
小程序 → 本地存储 → 数据丢失

目标架构（前后端分离）
小程序 → API → 后端 → 数据库
管理后台 → API → 后端 → 数据库
```

### 2. 技术栈

```
前端层
├─ 小程序客户端: uni-app + Vue 3
└─ 管理后台: Vue 3 + Element Plus

应用层
├─ 后端服务: Spring Boot 3
├─ ORM框架: MyBatis Plus
└─ 认证: JWT

数据层
├─ 数据库: MySQL 8.0
└─ 缓存: Redis 7.0
```

### 3. API设计

```
RESTful API
├─ GET     /api/product/list      # 获取商品列表
├─ GET     /api/product/:id       # 获取商品详情
├─ POST    /api/cart/add          # 添加到购物车
├─ POST    /api/order/create      # 创建订单
└─ ...
```

## ⚠️ 注意事项

### 开发环境

- ✅ JDK 必须是 17 或更高版本
- ✅ MySQL 字符集必须是 utf8mb4
- ✅ 小程序要求 HTTPS（生产环境）

### 数据迁移

- ⚠️ 本地存储数据无法自动迁移
- ⚠️ 测试时需要清除旧的本地数据
- ⚠️ 生产环境上线前需要数据清理

### 兼容性

- ✅ 保留本地存储代码作为降级方案
- ✅ API失败时显示友好提示
- ✅ 做好错误处理
