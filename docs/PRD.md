# 产品需求文档（PRD）

> 藏药商城小程序 · 版本 v2.0 · 2025年

---

## 一、产品概述

### 1.1 产品定位

藏药商城小程序是一个面向 C 端消费者的藏药电商平台，同时配套商家端管理后台。用户通过微信小程序浏览、购买正宗藏药产品；商家通过 Web 后台管理商品、订单和用户。

### 1.2 目标用户

| 角色 | 描述 |
|---|---|
| **消费者** | 关注健康养生、对藏药有兴趣的微信用户 |
| **商家管理员** | 负责运营的商家，管理商品、处理订单 |

### 1.3 核心价值

- 为消费者提供安全、便捷的藏药购买渠道
- 为商家提供订单管理和商品运营工具
- 通过"藏药知识"模块传递藏药文化，增强用户信任感

---

## 二、功能模块

### 2.1 小程序端（消费者）

#### 2.1.1 首页

- **轮播图**：展示活动/商品海报（数据来源：`/banner/list`）
- **热销推荐**：展示推荐商品（`/product/recommend`），点击跳转商品详情
- **藏药知识**：科普文章入口，点击跳转知识详情页

**验收标准**：
- 首页加载完成时间 < 2s
- 轮播图自动播放，间隔 3s
- 推荐商品图片使用本地图片映射，避免外链失效

#### 2.1.2 商品列表

- 全部商品展示（仅展示 `status=1` 上架商品）
- 分类筛选（顶部 Tab）
- 搜索功能（关键词匹配商品名）
- 下拉刷新、上拉加载更多（分页，每页 10 条）

**数据字段**：商品名称、价格、销量、图片、分类

#### 2.1.3 商品详情

- 商品图片（本地映射）
- 商品名称、价格、库存
- 商品描述（富文本）
- 规格选择（数量）
- 操作：加入购物车、立即购买

#### 2.1.4 购物车

- 商品列表（读取本地 localStorage `cartList`）
- 数量调整（±1）
- 单选/全选
- 底部合计金额、去结算按钮
- 为您推荐（推荐商品，调用真实接口）

**注意**：购物车数据存于本地，不与后端同步，不受登录状态影响。

#### 2.1.5 结算与下单

- 收货地址选择/新增（调用 `/address/list`）
- 商品清单展示
- 配送方式（固定：快递）
- 费用明细（商品金额 + 运费）
- 提交订单（`POST /order/create`）

**下单数据结构**：
```json
{
  "addressId": 1,
  "items": [{ "productId": 1, "quantity": 2, "price": 299, "productName": "冬虫夏草" }],
  "totalAmount": 598,
  "remark": ""
}
```

#### 2.1.6 订单管理

**订单列表**：
- Tab 筛选：全部、待付款、待发货、待收货、已完成
- 每个订单展示：商品图片+名称、数量、金额、状态、操作按钮

**订单操作**：

| 操作 | 触发状态 | 接口 |
|---|---|---|
| 去支付 | 待付款（0） | `POST /order/pay/{id}` → 状态变为1 |
| 取消订单 | 待付款（0） | `POST /order/cancel/{id}` → 状态变为4 |
| 确认收货 | 待收货（2） | `POST /order/confirm/{id}` → 状态变为3 |
| 删除订单 | 已完成/已取消 | `POST /order/delete/{id}` |

**订单详情**：
- 收货信息、物流信息（发货后显示）
- 商品明细、费用明细
- 订单编号、下单时间、支付时间

#### 2.1.7 个人中心

- 用户头像、昵称（默认头像：`static/images/avatar-default.jpg`）
- 订单快捷入口（待付款、待发货、待收货、已完成）
- 功能入口：收货地址、优惠券、设置

#### 2.1.8 收货地址

- 地址列表（默认地址置顶）
- 新增/编辑地址（姓名、手机号、省市区、详细地址、是否默认）
- 删除地址

#### 2.1.9 藏药知识

- 文章详情页（标题、内容、图片）
- 从首页入口进入

---

### 2.2 商家管理后台（adminWeb）

访问地址：`http://localhost:8080/api/admin-web/index.html`（后端启动后生效）

#### 2.2.1 登录

- 输入框：管理秘钥
- 验证方式：`POST /admin/login { secretKey }`
- 登录后 token 存入 localStorage，页面刷新自动验证有效性

#### 2.2.2 数据概览（Dashboard）

- 统计卡片：总订单数、总销售额、商品数、用户数
- 最近订单列表（最新 5 条）
- 热销商品列表（销量前 5）

#### 2.2.3 商品管理

| 功能 | 说明 |
|---|---|
| 商品列表 | 展示所有商品（含下架），支持分页和关键词搜索 |
| 新增商品 | 填写名称、价格、库存、分类、描述 |
| 编辑商品 | 修改商品信息 |
| 上架/下架 | 切换商品 status（0↔1） |

#### 2.2.4 订单管理

| 功能 | 说明 |
|---|---|
| 订单列表 | 所有订单，支持状态筛选、关键词搜索、分页 |
| 订单详情 | 弹窗显示完整订单信息 |
| 发货 | 填写快递公司和快递单号，状态从1→2 |
| 修改状态 | 手动修改订单状态 |

#### 2.2.5 用户管理

| 功能 | 说明 |
|---|---|
| 用户列表 | 用户信息、注册时间、消费金额，支持分页 |
| 封禁/解封 | 切换用户 status（0↔1） |
| 用户详情 | 弹窗查看详细信息 |

## 三、状态枚举

### 订单状态

| 值 | 含义 | 可进行的操作 |
|---|---|---|
| `0` | 待付款 | 支付（→1）、取消（→4） |
| `1` | 待发货 | 发货（→2，商家操作） |
| `2` | 待收货 | 确认收货（→3） |
| `3` | 已完成 | 删除 |
| `4` | 已取消 | 删除 |

### 商品状态

| 值 | 含义 |
|---|---|
| `0` | 下架（用户端不可见） |
| `1` | 上架（正常展示） |

### 用户状态

| 值 | 含义 |
|---|---|
| `0` | 正常 |
| `1` | 禁用（登录时拒绝） |

## 四、非功能需求

| 需求 | 要求 |
|---|---|
| 接口响应时间 | 普通查询 < 500ms |
| 图片加载 | 本地图片优先，避免外链图片失效 |
| 鉴权安全 | 小程序用 JWT（7天有效）；后台用固定秘钥 |
| 跨域 | 后端配置 CORS，允许小程序域名和 localhost 访问 |
| 数据完整性 | 下单时校验库存，订单创建后自动记录时间戳 |

---

## 五、已知限制（当前版本）

- 支付为模拟支付（点击"去支付"后直接将订单状态更新为1），未接入真实微信支付
- 图片上传为 Mock（未接入 OSS），商品图片使用本地预设图片
- Redis 缓存当前禁用，使用内存缓存
- 微信登录使用测试账号（`POST /user/test-login`），未获取真实 openId
