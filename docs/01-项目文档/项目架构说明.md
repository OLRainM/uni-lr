# 藏药小程序项目架构说明

## 项目概述

藏药小程序是一个完整的电商系统，包含前端小程序和后端API服务。

## 技术架构

### 前端技术栈

- **框架**: uni-app (Vue 3)
- **状态管理**: Vuex
- **UI组件**: uni-ui
- **样式**: SCSS
- **平台**: 微信小程序、H5

### 后端技术栈

- **开发语言**: Java 17
- **框架**: Spring Boot 3.2.0
- **ORM**: MyBatis Plus 3.5.5
- **数据库**: MySQL 8.0
- **缓存**: Redis
- **认证**: JWT
- **构建工具**: Maven

## 系统架构

### 三层架构

```
┌─────────────────┐
│  前端层（小程序） │
│   uni-app + Vue  │
└────────┬────────┘
         │ HTTP/HTTPS
         │ RESTful API
┌────────┴────────┐
│    后端层（API）  │
│  Spring Boot     │
│  - Controller    │
│  - Service       │
│  - Repository    │
└────────┬────────┘
         │ MyBatis Plus
┌────────┴────────┐
│    数据层        │
│  MySQL + Redis   │
└─────────────────┘
```

## 核心模块

### 1. 用户模块
- 微信登录授权
- 用户信息管理
- JWT Token认证

### 2. 商品模块
- 商品分类管理
- 商品列表查询（分页、筛选、排序）
- 商品详情展示

### 3. 购物车模块
- 购物车商品管理
- 数量调整
- 商品选择

### 4. 订单模块
- 订单创建
- 订单查询（状态筛选）
- 订单状态流转
- 订单取消/确认

### 5. 地址模块
- 收货地址CRUD
- 默认地址设置

### 6. 知识模块
- 藏药知识展示
- 文章阅读统计

## 数据库设计

### 主要表结构

1. **tb_user** - 用户表
2. **tb_category** - 商品分类表
3. **tb_product** - 商品表
4. **tb_cart** - 购物车表
5. **tb_order** - 订单表
6. **tb_order_item** - 订单明细表
7. **tb_address** - 收货地址表
8. **tb_knowledge** - 藏药知识表

详细表结构见 `backend/src/main/resources/sql/tibetan_medicine.sql`

## API设计

### RESTful API规范

- 使用标准HTTP方法：GET、POST、PUT、DELETE
- URL命名使用名词复数形式
- 统一使用JSON格式
- 统一响应格式

### 统一响应格式

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 接口模块

- `/api/user` - 用户相关接口
- `/api/product` - 商品相关接口
- `/api/cart` - 购物车相关接口
- `/api/order` - 订单相关接口
- `/api/address` - 地址相关接口
- `/api/knowledge` - 知识相关接口

详细接口文档见 `backend/API接口文档.md`

## 安全设计

### 认证授权

- 使用JWT Token进行用户认证
- Token有效期7天
- 请求头携带：`Authorization: Bearer {token}`

### 数据安全

- 使用HTTPS加密传输
- 敏感数据加密存储
- SQL注入防护（MyBatis Plus参数化查询）
- XSS攻击防护

### 权限控制

- 用户登录状态验证
- 资源访问权限控制
- 数据隔离（用户只能访问自己的数据）

## 性能优化

### 前端优化

- 图片懒加载

- 数据缓存（Vuex）

## 开发规范

### 代码规范

- 遵循阿里巴巴Java开发手册
- 使用Lombok简化代码
- 统一异常处理
- 日志规范（SLF4J）

### 命名规范

- 实体类：使用名词，首字母大写
- 接口：使用动词，首字母小写
- 变量：驼峰命名
- 常量：全大写，下划线分隔

### 注释规范

- 类级别注释说明功能
- 方法注释说明参数和返回值
- 复杂逻辑添加行内注释
