## 🎯 开发流程

### Step 1: 启动后端服务

#### 方法 A：在 IDEA 中启动
1. 打开 `backend` 项目
2. 运行 `TibetanMedicineApplication.java`
3. 看到 `Started TibetanMedicineApplication` 表示成功
4. 访问测试：http://localhost:8080/api/product/list

#### 方法 B：使用 Maven 命令
```powershell
cd e:\project\uniapp-lr\backend
mvn spring-boot:run
```

**启动成功标志**：
```
Tomcat started on port(s): 8080 (http) with context path '/api'
```

---

### Step 2: 启动小程序前端

```powershell
# 进入项目根目录
cd e:\project\uniapp-lr

# 启动开发模式
npm run dev:mp-weixin
```

**编译成功标志**：
```
DONE  Build complete. Watching for changes...
```

**输出目录**：`dist/dev/mp-weixin/`

2. 点击 **+** 创建项目

4. 点击**确定**

#### 后续开启
直接在微信开发者工具中打开项目即可

## 🐛 常见问题

### Q1: 小程序白屏，没有数据

**检查**：
1. 后端是否启动成功？
   - 访问：http://localhost:8080/api/product/list
   - 应该能看到 JSON 数据

2. 查看微信开发者工具的 **Console**
   - 是否有网络请求错误？
   - 是否有 JavaScript 错误？

3. 查看 **Network** 标签
   - API 请求是否成功（状态码 200）
   - 响应数据是否正确

**解决**：
```powershell
# 重新启动后端
cd backend
mvn spring-boot:run

# 重新编译前端
cd ..
npm run dev:mp-weixin
```

---

### Q2: 提示"不在以下 request 合法域名列表中"

**原因**：微信小程序默认不允许请求 localhost

**解决**：
1. 在微信开发者工具右上角
2. 点击**详情**
3. 勾选**不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书**

---

### Q3: 修改代码后不生效

**前端代码修改**：
- 保存后自动重新编译
- 在开发者工具中点击**编译**按钮

**后端代码修改**：
- 在 IDEA 中重新运行项目
- 或使用 Spring Boot DevTools 热重载

---

### Q4: npm run dev:mp-weixin 报错

**检查 Node.js 版本**：
```powershell
node -v  # 应该是 v16 或更高
```

**重新安装依赖**：
```powershell
rm -rf node_modules package-lock.json
npm install
npm run dev:mp-weixin
```

## 🚀 准备部署时

当本地开发完成，准备部署到服务器时：

### 后端部署步骤
1. 打包项目：`mvn clean package`
2. 上传到服务器：`target/tibetan-medicine-api-1.0.0.jar`
3. 在服务器运行：`java -jar tibetan-medicine-api-1.0.0.jar`

### 前端配置修改
修改 `src/utils/request.js`：
```javascript
// 本地开发
const BASE_URL = 'http://localhost:8080/api'

// 改为服务器地址
const BASE_URL = 'http://115.190.125.177:8080/api'
// 或使用域名
const BASE_URL = 'https://api.yourdomain.com/api'
```

### 重新编译小程序
```powershell
npm run build:mp-weixin
```

上传 `dist/build/mp-weixin` 到微信小程序平台
