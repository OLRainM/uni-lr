## 💡 核心概念

### 当前架构（本地存储）

```
小程序 → 本地存储（手机）
         ↓
   数据会丢失，商家看不到
```

**问题**：
- ❌ 卸载小程序数据丢失
- ❌ 换手机无法同步
- ❌ 商家无法管理订单

### 目标架构（前后端分离）

```
小程序 ────┐
           ├→ API → 后端服务 → 数据库
管理后台 ──┘
```

**优势**：
- ✅ 数据持久化
- ✅ 多端同步
- ✅ 商家可管理
