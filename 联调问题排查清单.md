# 藏药小程序联调问题排查清单（Windows版）

> 目标：解决小程序 `ERR_CONNECTION_REFUSED`、接口 4xx/5xx、数据库报错等联调问题。

## 一、先确认运行环境

- [ ] Java 17 可用（`java -version`）
- [ ] Python 可用（`python --version`，用于辅助脚本）
- [ ] MySQL 可连接
- [ ] Redis 可选（本项目可先不依赖核心流程）

---

## 二、后端服务可用性（第一优先级）

### 1) 启动后端
- [ ] 在 IDE 运行 `TibetanMedicineApplication`
- [ ] 观察控制台出现：`Tomcat started on port 8080`

### 2) 检查 8080 端口监听
```powershell
Get-NetTCPConnection -LocalPort 8080 -State Listen
```
- [ ] 有结果（说明服务已监听）

### 3) 健康检查
浏览器访问：
- [ ] `http://localhost:8080/api/health`
- [ ] `http://localhost:8080/api/`

---

## 三、数据库一致性（第二优先级）

> 若日志出现 `tb_xxx doesn't exist`，说明表名与代码不一致。

### 1) 执行数据库修复脚本（推荐）
- [ ] 执行：`backend/src/main/resources/sql/fix_database.sql`

或使用脚本：
- [ ] `python backend/run_fix_database.py`

### 2) 验证核心表
- [ ] `users`
- [ ] `categories`
- [ ] `products`
- [ ] `knowledge`
- [ ] `banners`

---

## 四、前端请求配置核对（第三优先级）

### 1) BASE_URL 核对
当前默认：`http://localhost:8080/api`

### 2) 切换环境（已支持）
可在小程序启动前设置：
```javascript
uni.setStorageSync('API_BASE_URL', 'http://你的IP:8080/api')
```
- [ ] 本机调试用 localhost
- [ ] 真机调试改为局域网 IP

---

## 五、核心接口联调顺序（建议）

按顺序调用，逐步放量：

1. 公共接口：
- [ ] `GET /banner/list`
- [ ] `GET /product/categories`
- [ ] `GET /knowledge/list`
- [ ] `GET /product/recommend`

2. 登录接口：
- [ ] `POST /user/login`

3. 需要登录态：
- [ ] `GET /user/info`（带 token）
- [ ] `POST /user/update`（带 token）

---

## 六、出现问题时的快速定位

### A. `ERR_CONNECTION_REFUSED`
- [ ] 80% 是后端没启动/端口未监听
- [ ] 检查端口、健康接口、启动日志

### B. 401 未授权
- [ ] 登录接口是否返回 token
- [ ] 请求头是否携带 `Authorization`
- [ ] token 是否过期

### C. 500 服务器异常
- [ ] 看 `backend/logs/tibetan-medicine.log`
- [ ] 优先检查 SQL 报错、表不存在、字段不匹配
